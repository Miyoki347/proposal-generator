プロジェクト概要

#overview
目的

日本のクラウドソーシングプラットフォーム（CrowdWorks・ランサーズ）向けの提案文ドラフトをAIが自動生成するツール。フリーランサーが案件に応募する際の提案文作成の手間を削減し、プロフィール情報を活かした説得力のある文章を迅速に生成することを目的とする。

対象ユーザー

CrowdWorks・ランサーズで案件受注を行うフリーランサー。職種は問わないが、初期実装ではWebエンジニア・デザイナー・ライター・マーケターなど12職種のプリセットを用意している。

将来の展開

現在は個人利用の無料ツール。将来的にはFirebase App Hostingで有料SaaSとして展開予定。Vercel（テスト）→ Firebase（本番）の二段構成でデプロイ。

UserProfile

interface UserProfile {
  name:               string;   // 提案文の名乗りに使用
  jobType:            string;   // JOB_TYPES[].value と対応
  experienceYears:    string;   // EXPERIENCE_YEARS[].value と対応

  achievementSummary: string;   // 実績サマリー（数字を含む文章）
  strengths:          string;   // 強み・差別化ポイント
  portfolio:          PortfolioItem[];
  skillPresets:       SkillPreset[]; // 保存済みスキルセット
}

interface PortfolioItem {
  id:          string;
  title:       string;
  url?:        string;
  description: string;
  tags:        string[]; // AI選定に使うキーワード
}

interface SkillPreset {
  id:                 string;   // Date.now().toString()
  nickname:           string;   // ユーザーが付けた名前
  jobType:            string;
  experienceYears:    string;
  skills:             string[];
  achievementSummary: string;
}

生成の基本方針

#generation
R-01
数字を必ず含める — 実績や経験を語るとき、入力された数字（件数・評価・年数・達成率）を具体的に文中に盛り込む。「多数の実績があります」は NG。
R-02
案件の課題に言及する — 案件概要を読み解き、クライアントが解決したい課題・ゴールを自分の言葉で言い換えた上で、自分がそれを解決できる根拠を示す。
R-03
ポートフォリオは案件関連のものだけ選ぶ — 全作品を列挙せず、案件のタグ・キーワードと合致するものだけを引用する。最大3件まで。
R-04
強みは差別化として機能させる — 「丁寧に対応します」「コミュニケーションを大切にします」のような凡庸な表現は避け、profile.strengths の具体内容をそのまま活用する。
R-05
類似実績は冒頭近くに配置する — similarExperience が入力されている場合、それを証拠として早めに提示する。信頼性の確立は提案文の前半で行う。
R-06
文字数は400〜600字を目安にする — 長すぎる提案は読まれない。簡潔に核心を伝える。ポートフォリオ引用を含めても700字以内を上限とする。
R-07
架空の情報を生成しない — 入力されていない実績・スキル・URLは生成してはならない。不足している情報があれば省略するか一般的な表現にとどめる。

プラットフォーム別の提案構成

#platforms
CrowdWorks
CW-1
書き出し：「はじめまして、{name} と申します。」から始める。名前が未設定の場合は「はじめまして。」
CW-2
構成順：自己紹介（職種・経験年数）→ 実績・スキル証明 → ポートフォリオ引用 → 本案件への対応方針 → 納期・見積もり言及 → 締め
CW-3
文体：敬語（です・ます調）を基本とする。砕けすぎず、フォーマル寄りの文体。
CW-4
締め：「ご検討のほど、よろしくお願いいたします。」など、控えめに検討を促す表現で終わる。
ランサーズ
LC-1
書き出し：クライアントの悩みや課題への共感から入る。例：「〜でお困りではないでしょうか。」「〜を実現したいというご要望、拝見しました。」
LC-2
構成順：共感フレーズ → 解決策の提示 → 自分が解決できる根拠（実績・スキル） → ポートフォリオ引用 → 具体的なアプローチ → 次のアクション促し
LC-3
実績の見せ方：数字を積極的に前面に出す。「〇件の実績」「評価 ★4.8」「納期遵守率100%」のように冒頭近くで提示する。
LC-4
締め：「ぜひ一度ご相談ください。」「まずはご要件をお聞かせいただければと思います。」など、次のアクションを促す積極的な表現で終わる。

トーン定義

#tones
トーン値 説明 向いているケース
polite 丁寧語・敬語を徹底。信頼感と誠実さを前面に出す文体。感情表現は控えめ。 初めてのクライアント、高単価案件、専門性重視の案件
business 要点を絞ったプロフェッショナルな文体。回りくどい表現を省き、価値を端的に伝える。 企業クライアント、継続案件、要件が明確な案件
passionate 熱意・やる気・この案件をやりたいという気持ちを前面に出す。感情的な言葉を適切に使う。 競争が激しい案件、スタートアップ、実績より意欲を見る案件
実装上の注意
トーンはプロンプト内で文体指示として渡す。「丁寧」= 敬語徹底・感情控えめ、「ビジネス」= 簡潔・価値提示優先、「熱意重視」= 積極的表現・気持ちを言語化。プラットフォームとトーンの組み合わせで合計6パターンの文体バリエーションが生まれる。

プロフィール情報の活用方針

#profile-usage
フィールド プロンプトでの使われ方
name CWの書き出し「はじめまして、〇〇と申します」に使用。未設定時は名乗り省略。
jobType + experienceYears 「〇〇として〇年の経験を持つ」という実績の根拠として使用。
skills 案件に直接関係するスキルのみを抽出して言及する。全スキルを列挙しない。
achievementSummary 信頼性の証拠として、特に数字部分を強調して提案文に組み込む。
strengths 競合フリーランサーとの差別化として、提案の後半〜締め前に配置する。
portfolio タグと案件概要を照合し、関連度の高い作品を最大3件ピックアップして引用する。URLが設定されている場合は記載する。
similarExperience 「同種の実績があること」の証拠として、早い段階（2〜3文目）で明示する。

品質チェック項目

#quality
OK
案件タイトルまたは概要のキーワードが提案文中に自然な形で盛り込まれている
OK
実績・経験を裏付ける具体的な数字が1つ以上含まれている
OK
選択したプラットフォームの構成ルールに沿った文章になっている
OK
ポートフォリオを引用する場合、案件と関係ない作品は含まれていない
OK
文字数が400〜700字の範囲に収まっている
OK
選択されたトーンに合った文体・語尾で統一されている

NGパターン

#ng
NG

NG
「丁寧に対応します」「コミュニケーションを大切にします」のような汎用的な差別化表現
NG

NG

NG

NG

NG
